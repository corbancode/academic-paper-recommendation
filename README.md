# Academic Paper Recommendation

MSc project repository for building and evaluating an academic paper recommendation system.  
This repo currently focuses on the technical implementation, starting with data collection from arXiv.

## Data Collection (arXiv)

The notebook `arxiv-data-collection.ipynb` collects papers from selected categories (`cs.LG`, `cs.CL`, `cs.AI`, `stat.ML`) using the arXiv API, applies date filtering, deduplicates records, and writes:
- `data/papers.jsonl`
- `data/dataset_metadata.json`

The script handles arXiv pagination limits by switching sort order when needed (descending to ascending) so larger datasets can be collected safely.

## OpenAlex Enrichment

The notebook `openalex_enrich.ipynb` enriches collected papers with OpenAlex metadata using DOI-first matching and title-based fallback.

It writes:
- `data/openalex_enriched.jsonl`
- `data/openalex_enrichment_metadata.json`

API failures are handled gracefully: records that still fail after retries are skipped and marked in the output.

## Query Builder

The notebook `build-queries.ipynb` constructs the evaluation query set and relevance judgments from the enriched corpus.

It writes:
- `data/queries.jsonl`
- `data/qrels.jsonl`
- `data/query_qrels_metadata.json`

## Evaluation Notebook Scope

`academic-paper-recommendation.ipynb` does not collect or enrich raw data directly.  
It uses outputs generated by:
- `arxiv-data-collection.ipynb`
- `openalex_enrich.ipynb`
- `build-queries.ipynb`

Inside `academic-paper-recommendation.ipynb`, the implementation and evaluation workflow covers:
- data loading
- data exploration
- data cleaning
- preprocessing/indexing
- embedding generation
- recommendation evaluation with standard IR metrics
- embedding geometry analysis

## Important Reproducibility Guidance

For consistent evaluation results in `academic-paper-recommendation.ipynb`, do **not** re-run:
- `arxiv-data-collection.ipynb`
- `openalex_enrich.ipynb`
- `build-queries.ipynb`

These upstream notebooks can regenerate dataset/query over time (new submissions, metadata updates, API ordering effects) artifacts and change the final metrics and conclusions.
